# Configuration

## Compiler Options

The URUS compiler (`urusc`) is configured through command-line flags. There is no separate config file.

### CLI Flags

| Flag | Default | Description |
|------|---------|-------------|
| `--help`, `-h` | - | Show help message |
| `--version`, `-v` | - | Show compiler version |
| `--emit-c` | off | Output generated C to stdout without compiling |
| `--tokens` | off | Display lexer tokens |
| `--ast` | off | Display the Abstract Syntax Tree |
| `-o <name>` | `a.exe` / `a.out` | Output binary name |

### Examples

```bash
# Default: compile to binary
urusc program.urus

# Custom output name
urusc program.urus -o myapp

# Only generate C (for debugging)
urusc --emit-c program.urus

# View tokens (for debugging lexer)
urusc --tokens program.urus

# View AST (for debugging parser)
urusc --ast program.urus

# Show version
urusc --version
```

## Environment Variables

URUS does not use environment variables directly. However, the compiler depends on:

| Variable | Used By | Notes |
|----------|---------|-------|
| `PATH` | OS | Must contain the location of `gcc`. On Windows, the compiler auto-detects GCC in common paths (MSYS2, MinGW, WinLibs) and injects into PATH |

## Build Configuration

### CMake (Recommended)

```bash
cd compiler
cmake -S . -B build
cmake --build build --config Release
```

#### CMake Options

| Variable | Default | Description |
|----------|---------|-------------|
| `CMAKE_BUILD_TYPE` | `Release` | Build type: `Debug`, `Release`, `RelWithDebInfo` |
| `PROJECT_VERSION` | `0.2.2` | Compiler version number |
| `PROJECT_VERSION_TAGS` | `(F)-Fixed` | Version tag string |

#### Platform-specific Notes

| Platform | Generator | Notes |
|----------|-----------|-------|
| Windows (MSVC) | Visual Studio | Adds `_CRT_SECURE_NO_WARNINGS`, `_CRT_NONSTDC_NO_DEPRECATE` |
| Windows (MinGW) | MinGW Makefiles | Uses GCC directly |
| Linux | Unix Makefiles | Default GCC |
| macOS | Unix Makefiles | Uses system GCC or Homebrew GCC |

### Debug vs Release

| Mode | CMake | Notes |
|------|-------|-------|
| Debug | `-DCMAKE_BUILD_TYPE=Debug` | Includes debug symbols, no optimization |
| Release | `-DCMAKE_BUILD_TYPE=Release` | Optimized with `-O2` |

## Generated C Configuration

The C files generated by URUS use:
- **Standard:** C11 (`-std=c11`)
- **Math library:** `-lm` (for `sqrt`, `abs`, etc.)
- **Optimization:** `-O2`
- **Runtime:** `urus_runtime.h` (embedded in generated C automatically)

### Temp File

The compiler writes a temporary C file `_urus_tmp.c` in the working directory, then deletes it after GCC finishes. This file can be inspected with `--emit-c` if compilation fails.

## Import Path Resolution

Imports are resolved relative to the location of the importing file:

```urus
// If this file is at /project/src/main.urus
import "utils.urus";       // -> /project/src/utils.urus
import "lib/math.urus";    // -> /project/src/lib/math.urus
```

- No search paths or library paths
- Circular imports are detected and produce an error
- Maximum 64 imports per program
- Imported files are merged into a single generated C file

## GCC Detection (Windows)

On Windows, the compiler searches for GCC in the following paths:

1. `C:/msys64/mingw64/bin/gcc.exe` (MSYS2)
2. `C:/mingw64/bin/gcc.exe` (Standalone MinGW)
3. `C:/Program Files/mingw-w64/*/mingw64/bin/gcc.exe`
4. System PATH

When found, the compiler injects the GCC directory into PATH to ensure `cc1` and other GCC tools are accessible.
